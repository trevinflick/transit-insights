# ============================================================================
# COTA-INSIGHTS cron block — source of truth for the live crontab.
# ============================================================================
#
# Install with:   scripts/install-crontab.sh
# (safe marker-merge — replaces only this block, never your other cron jobs.)
#
# Bus detection, recaps, health audits, reference-data refresh, and backups
# all run here. Single agency (COTA, bus-only) — no multi-mode staggering is
# needed; offsets below only avoid jobs colliding with each other.
# The /path/to/cota-insights placeholder is filled in with the repo checkout at
# install (cron does NOT expand variables in R= assignment lines, so the path
# must be inlined — the install script does the substitution).
# Logs go to cron/<name>-cron.log (gitignored).

# COTA-INSIGHTS-START

# All jobs run via bin/cron-run.sh <log-name> <script> [args...] which handles
# cd, timestamping, and log redirection.
R=/path/to/cota-insights/bin/cron-run.sh

# --- Buses ---
# cross-route pileup (2+ routes at one corner). Runs 1 min BEFORE bus-bunching
# so a posted multi-route pileup suppresses the per-route post for the same
# buses (recentCrossBunchMemberIds, 10-min window).
0-59/20 *  *  *  *   $R bus-cross-bunching bin/bus/cross-bunching.js
1-59/20 *  *  *  *   $R bus-bunching     bin/bus/bunching.js
6-59/20 *  *  *  *   $R bus-gaps         bin/bus/gaps.js
# bus-thin-gaps: gap detection for low-frequency routes, every 15 min on a :13
# offset so it doesn't share a wall minute with bus-gaps (:06/26/46).
13-58/15 *  *  *  *  $R bus-thin-gaps    bin/bus/thin-gaps.js
0       */2 *  *  *  $R bus-speedmap     bin/bus/speedmap.js
7       *  *  *  *   $R bus-ghosts       bin/bus/ghosts.js
# observe-buses runs every minute. COTA's GTFS-realtime feeds are unauthenticated
# polled protobuf endpoints (no daily-request cap like CTA's BusTime), so cadence
# here is about freshness, not quota. bunching/gaps/pulse all read the cache
# (maxStaleMs=90s) so observe-buses is the only poll site for the all-routes
# workload.
*       *  *  *  *   $R observe-buses    scripts/observeBuses.js

# --- Recaps ---
20      10 *  *  0   $R bus-recap        bin/bus/recap.js   --window=week
30      10 1  *  *   $R bus-recap        bin/bus/recap.js   --window=month

# --- Health audits ---
*/30    *  *  *  *   $R audit-alerts     bin/audit-alerts.js

# --- Reference data refresh ---
15      3  *  *  *   $R fetch-gtfs       scripts/fetch-gtfs.js

# --- Backups ---
# Off-box backup of state/history.sqlite to Cloudflare R2. Standalone shell
# script (not via cron-run.sh, which is node-only); see docs/BACKUPS.md. The
# /path/to/cota-insights placeholders are substituted by the install procedure
# above alongside the R= line.
17      4  *  *  *   /path/to/cota-insights/scripts/backup-db.sh >> /path/to/cota-insights/cron/backup-db-cron.log 2>&1

# COTA-INSIGHTS-END
